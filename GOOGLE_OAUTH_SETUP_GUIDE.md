# Google OAuth 認証セットアップガイド（管理者向け）

このガイドでは、Google をログイン方法として使えるようにするための初期設定を、管理者向けにわかりやすく説明します。

一般ユーザーは、このガイドの設定作業を行う必要はありません。
一般ユーザーは、管理者が設定を完了したあとに、ログイン画面で Google でログインするだけです。

このガイドは、Google OAuth を本番公開で運用する前提で記載しています。

---

## 📋 準備

以下のものを用意してください。
- ブラウザ（Chrome、Firefox など）
- Google アカウント（Gmail など）
- Supabase プロジェクト ID（アプリの URL から取得）

**Supabase プロジェクト ID の確認方法:**
1. https://supabase.com にアクセス
2. あなたのプロジェクトを開く
3. URL を見る（例：`https://app.supabase.com/project/gnnfzimhfizbfhvwuzbf`）
4. `gnnfzimhfizbfhvwuzbf` がプロジェクト ID です

---

## 🚀 手順1：Google Cloud Console にログイン

1. ブラウザで https://console.cloud.google.com を開く
2. 右上の「ログイン」をクリック
3. あなたの Google アカウントでログイン
4. 言語が英語の場合は、左上の歯車 → 言語を「日本語」に変更

---

## 📁 手順2：プロジェクトを作成

1. 画面上部に「プロジェクトを選択」という青いボタンがあります
2. それをクリック
3. 右上に「新規プロジェクト」ボタンが出ます
4. クリック
5. 以下を入力：
   - **プロジェクト名**: 「南国市議会ポータル」など、自由に入力
6. 「作成」をクリック
7. 完了するまで 1〜2 分待ちます（右上に通知が出ます）

---

## 🔧 手順3：API を有効化

1. 左側のメニューから「API とサービス」をクリック
2. 「ライブラリ」をクリック
3. 検索ボックスに「Google+ API」と入力
4. 検索結果の「Google+ API」をクリック
5. 「有効にする」という青いボタンをクリック
6. 完了するまで少し待ちます

---

## 🎨 手順4：OAuth 同意画面を設定

### 4-1. OAuth 同意画面ページを開く

1. 左側のメニューから「API とサービス」をクリック
2. 「OAuth 同意画面」をクリック

### 4-2. ユーザータイプを選択

1. 「ユーザーの種類」で「外部」を選択
2. 「作成」をクリック

### 4-3. 情報を入力

以下の画面で必須項目を入力します：

**アプリ名:** 
- 「南国市議会ポータル」など、わかりやすい名前を入力

**ユーザーサポートメール:**
- あなたのメールアドレスを入力（Gmail など）

**デベロッパー連絡先メール（下部）:**
- あなたのメールアドレスを入力

4. 「保存して次へ」をクリック

### 4-4. スコープ画面

ここでは何もせず、そのまま「保存して次へ」をクリック

### 4-5. 公開ステータスを本番にする

1. OAuth 同意画面の設定を保存します
2. 公開ステータスを確認します
3. 「テスト中」になっている場合は「本番環境で公開」または「Publish App」を選びます
4. 確認画面が出たら内容を確認して確定します

**重要:**
- 本番公開にすると、テストユーザーの個別登録は不要になります
- 今回はカレンダー権限を止め、Google ログインに必要な最小権限だけで運用します

---

## 🔐 手順5：OAuth クライアント ID を作成

### 5-1. 認証情報ページを開く

1. 左側のメニューから「API とサービス」をクリック
2. 「認証情報」をクリック

### 5-2. クライアント ID を作成

1. 画面上部に「+ 認証情報を作成」というボタンがあります
2. それをクリック
3. 「OAuth クライアント ID」を選択

### 5-3. アプリケーションの種類を選択

1. 「ウェブアプリケーション」を選択
2. 「名前」に「Supabase」と入力

### 5-4. リダイレクト URI を追加

1. 「認可済みのリダイレクト URI」というセクションを探します
2. テキストボックスに以下を入力：
   ```
   https://gnnfzimhfizbfhvwuzbf.supabase.co/auth/v1/callback
   ```
   **注意:** `gnnfzimhfizbfhvwuzbf` は、あなたの Supabase プロジェクト ID に置き換えてください

3. 「作成」をクリック

### 5-5. Client ID と Secret をコピー

1. ポップアップが出ます
2. 以下の 2 つをコピーして、テキストファイルに保存してください：
   - **クライアント ID**
   - **クライアントシークレット**

```
🔒 保存例：
クライアント ID: 123456789-abcdefghijk.apps.googleusercontent.com
クライアントシークレット: GOCSPX-abcdefghijklmnopqrst
```

---

## 🌐 手順6：Supabase に設定

### 6-1. Supabase ダッシュボードを開く

1. https://supabase.com にアクセス
2. あなたのプロジェクトを開く

### 6-2. 認証設定ページを開く

1. 左側のメニューから「Authentication」をクリック
2. その中の「Providers」をクリック

### 6-3. Google プロバイダーを有効化

1. プロバイダー一覧から「Google」を探してクリック
2. トグルを「ON」にする

### 6-4. Client ID と Secret を貼り付け

1. **Client ID** の欄に、手順 5-5 でコピーしたクライアント ID を貼り付け
2. **Client Secret** の欄に、手順 5-5 でコピーしたクライアントシークレットを貼り付け
3. 「Save」をクリック

完了です！

---

## ✅ テスト

### ログイン画面で試す

1. アプリのログイン画面を開く
2. 「Google でログイン」ボタンをクリック
3. Google のログイン画面が出ます
   - 手順 4-5 で登録したメールアドレスでログイン
4. 「アクセスを許可」をクリック
5. アプリにログインできたら成功！

### テスト前の確認

以下の 2 つがそろっていないと、一般ユーザーはログインできません。

1. member_directory に、その人の Gmail アドレスが登録されている
2. Google Cloud Console の OAuth 同意画面が本番公開になっている

どちらか片方だけでは不十分です。

---

## 🚨 よくあるエラーと対処法

### ❌ エラー: 403 access_denied

**意味**: Google Cloud Console 側の OAuth 同意画面がまだテスト中のままです

**対処法**:
1. Google Cloud Console に戻る
2. 「OAuth 同意画面」を開く
3. 公開ステータスを確認する
4. 「テスト中」なら「本番環境で公開」に切り替える
5. 保存後、もう一度ログインしてください

**重要:**
- member_directory に登録済みでも、Google Cloud 側の公開ステータスがテスト中だと 403 になります
- このエラーはアプリ本体の不具合ではなく、Google OAuth の公開設定の問題です

### ❌ エラー: redirect_uri_mismatch

**意味**: リダイレクト URI が間違っています

**対処法**:
1. Supabase のプロジェクト ID が正しいか確認
2. URL 内の `gnnfzimhfizbfhvwuzbf` を、あなたのプロジェクト ID に置き換え
3. Google Cloud Console で再度確認して、正しく入力してください

### ❌ エラー: invalid_client

**意味**: Client ID または Secret が間違っています

**対処法**:
1. Google Cloud Console で Client ID と Secret を再度確認
2. Supabase でコピペミスがないか確認
3. 最後に「Save」を忘れずクリック

## 🆘 困ったときは

以下を確認してください：

1. **Google Cloud Console のプロジェクト ID が合っているか**
   - URL から確認: `https://console.cloud.google.com/project/YOUR_PROJECT_ID`

2. **Supabase のプロジェクト ID が合っているか**
   - Supabase URL から確認

3. **Client ID と Secret をコピペミスしていないか**
   - 一文字でも間違うと動きません

4. **リダイレクト URI が正確か**
   - `https://gnnfzimhfizbfhvwuzbf.supabase.co/auth/v1/callback`
   - 末尾のスラッシュ有無も確認

わからないことがあったら、このガイドを最初から読み直すか、開発者に連絡してください。
